# Plan : Section Photos GBP avec URLs SEO

## Objectif
Ajouter un onglet "Photos" dans la vue client pour uploader et publier des photos directement sur la fiche Google Business Profile. Chaque photo est stockée avec une URL SEO-friendly (même système que les visuels).

## Architecture

### 1. Base de données — Table `location_photos`

```sql
CREATE TABLE IF NOT EXISTS location_photos (
    id              INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    location_id     INT UNSIGNED NOT NULL,
    category        VARCHAR(50) DEFAULT 'ADDITIONAL',
    seo_keyword     VARCHAR(200) DEFAULT NULL,
    caption         VARCHAR(500) DEFAULT NULL,
    file_path       VARCHAR(500) NOT NULL,
    file_url        VARCHAR(500) NOT NULL,
    file_size       INT UNSIGNED DEFAULT NULL,
    width           INT UNSIGNED DEFAULT NULL,
    height          INT UNSIGNED DEFAULT NULL,
    status          ENUM('draft','published','failed') DEFAULT 'draft',
    google_media_name VARCHAR(255) DEFAULT NULL,
    error_message   TEXT DEFAULT NULL,
    published_at    DATETIME DEFAULT NULL,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_location (location_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

Catégories GBP : COVER, PROFILE, EXTERIOR, INTERIOR, PRODUCT, AT_WORK, FOOD_AND_DRINK, TEAMS, ADDITIONAL

### 2. API — `app/api/photos.php`

| Action | Description |
|---|---|
| `list` | Liste les photos d'une location (avec stats) |
| `upload` | Upload fichier photo, stocke avec URL SEO |
| `update` | Modifie catégorie, mot-clé SEO, légende |
| `delete` | Supprime photo (fichier + DB) |
| `publish` | Publie sur GBP via API Media |
| `bulk_delete` | Suppression groupée |

**Upload** : Le fichier est renommé selon le pattern SEO existant :
- `/media/{client-slug}/{seo-keyword-slug}/{id}.jpg`
- Sans mot-clé : `/media/{client-slug}/photos/{id}.jpg`
- L'image est optimisée (redimensionnée max 2048px, compressée JPEG 85%)

**Publish** : Appel GBP API v4 `POST /accounts/{acc}/locations/{loc}/media`
```json
{
    "mediaFormat": "PHOTO",
    "sourceUrl": "https://app.boustacom.fr/app/media/client-slug/keyword/42.jpg",
    "locationAssociation": { "category": "EXTERIOR" }
}
```
Réutilise `resolveGoogleLocationPath()` + pattern d'auth identique à `publishPostToGoogle()`.

### 3. Fonction PHP — `publishPhotoToGoogle()` dans `functions.php`

Même pattern que `publishPostToGoogle()` :
- Prend `$photo`, `$googleLocationId`, `$token`, `$googleAccountName`, `$dbLocationId`
- Résout le path avec `resolveGoogleLocationPath()`
- POST vers `/media` endpoint
- Retourne `['success' => bool, 'google_media_name' => string]`

### 4. JS Module — `APP.photos` dans `app.js`

**Méthodes :**
- `load(locationId)` — Fetch API + render
- `render(photos, stats, locationId)` — Grille de photos avec :
  - Grid responsive (3 colonnes desktop, 2 tablet, 1 mobile)
  - Chaque photo : thumbnail, catégorie badge, statut badge, mot-clé SEO
  - Actions : publier, modifier, supprimer
- `showUploadForm(locationId)` — Modal/sidePanel :
  - Zone drag & drop + input file (multiple)
  - Input mot-clé SEO (réutilisé pour toutes les photos du batch)
  - Select catégorie GBP
  - Input légende optionnelle
  - Upload séquentiel avec progress
- `publish(photoId, locationId)` — Publie une photo
- `toggleSelectMode()` / `bulkDelete()` — Mode sélection groupée

**UI :**
- Header avec compteurs (Total, Publiées, Brouillons)
- Bouton "Ajouter des photos" (primary)
- Filtre par catégorie
- Grille photos avec lazy loading
